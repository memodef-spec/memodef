# proposals/

Spec change proposals.

## Format

Per [CONTRIBUTING.md §3](../CONTRIBUTING.md#3-spec-proposals). Sections: Summary, Motivation, Proposed Change, Backward Compatibility, Conformance Tests, Alternatives Considered, Open Questions.

## Workflow

1. **Open** — file the proposal as a PR adding `proposals/<name>.md`
2. **Discuss** — open period for community/maintainer/strategist input
3. **Iterate** — refine based on feedback
4. **Decide** — strategist files a decision in `../decisions/proposal-<name>.md`
5. **Implement** (if accepted) — maintainer drafts the schema change as a follow-on PR
6. **Version bump** — accepted spec changes bump the memodef schema version per [SCHEMA.md](../SCHEMA.md#versioning)
